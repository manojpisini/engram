STEP 2

Notion Workspace

Connect your Notion workspace to store all engineering intelligence data. ENGRAM will create 23 databases automatically.
Starts with ntn_ — this is your Internal Integration Secret
▶ How to get this
  1. Go to notion.so/profile/integrations
  2. Click "+ New integration"
  3. Name it ENGRAM, select your workspace
  4. Click "Submit"
  5. Copy the Internal Integration Secret (starts with ntn_)
  6. Under Capabilities, enable: Read, Update, Insert content
⚠ You must also share your Notion page with the integration: open any page → ••• → "Add connections" → select ENGRAM
ENGRAM creates all 23 databases as children of this page
▶ How to get the Page ID
  1. In Notion, create a new blank page (or use any existing page)
  2. Click ••• (top right) → "Add connections" → select your ENGRAM integration
  3. Click "Share" (top right) → "Copy link"
  4. The link looks like: notion.so/Your-Page-Title-abc123def456...
  5. The last 32 hex characters (after the final dash) is your Page ID
⚠ The page MUST be shared with your ENGRAM integration, otherwise database creation will fail.
Tip: You can also get the ID from the URL bar — open the page in Notion, and copy the 32-character hex string from the URL. Format it as: xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx (with dashes) or without — both work.
✓ ENGRAM will create a styled page with cover image, icon, and all 23 databases nested inside your shared page.
STEP 3

GitHub Connection

Link your GitHub account to ingest webhooks from your repositories.
Starts with github_pat_ (fine-grained) or ghp_ (classic)
▶ How to get this
Fine-grained token (recommended):
  1. Go to github.com/settings/tokens (Fine-grained)
  2. Click "Generate new token"
  3. Name: ENGRAM, Expiration: 90 days
  4. Repository access: select repos you want to monitor
  5. Permissions: Contents (Read), Pull requests (Read), Issues (Read), Webhooks (Read & Write)
  6. Click "Generate token" and copy it
Classic token (alternative):
  1. Go to github.com/settings/tokens (Classic)
  2. Scopes: repo, read:org, admin:repo_hook
Comma-separated list of owner/repo names. Example: manojpisini/engram, manojpisini/ipl-viz
A random string used to verify webhook payloads via HMAC-SHA256
▶ How to generate & configure
Generate a secret:
  1. Run in terminal: openssl rand -hex 32 (Linux/macOS)
  2. Or PowerShell: -join((1..64)|%{'{0:x}'-f(Get-Random -Max 16)})
  3. Or use any password generator (32+ characters)
Configure in GitHub:
  1. Go to your repo → Settings → Webhooks → Add webhook
  2. Payload URL: http://YOUR_SERVER:3000/webhook/github
  3. Content type: application/json
  4. Secret: paste the same secret you enter here
  5. Events: select "Send me everything" or pick Push, PRs, Releases
⚠ The secret must match exactly in both ENGRAM and GitHub. If left empty, webhooks will accept unsigned payloads (not recommended for production).
STEP 4

AI Intelligence

Power the 9 analysis agents with Claude to generate engineering insights.
Starts with sk-ant- — this powers all 9 intelligence agents
▶ How to get this
  1. Go to console.anthropic.com/settings/keys
  2. Sign in or create an Anthropic account
  3. Click "Create Key"
  4. Name it ENGRAM and copy the key
⚠ You need API credits in your Anthropic account. ENGRAM uses claude-sonnet-4-20250514 by default. Each webhook event costs roughly $0.01–0.05 in API usage depending on payload size.
STEP 5

Server Configuration

Defaults work for local use. Only change these if deploying to a server or using a custom port.
127.0.0.1 = localhost only • 0.0.0.0 = accept from all networks
Default: 3000
▶ Localhost vs Server mode
Localhost mode (default): ENGRAM runs on your machine. GitHub can't reach localhost, so use the .github/workflows/engram-notify.yml GitHub Action to forward events from CI to your local instance.

Server mode: Deploy ENGRAM on a VPS/cloud server with a public IP or domain. Set host to 0.0.0.0 and configure your GitHub webhook URL as http://your-server:3000/webhook/github. Use a reverse proxy (nginx/caddy) for HTTPS.

What ENGRAM Does

ENGRAM deploys 9 autonomous AI agents that continuously monitor your engineering workflow. GitHub PRs, CI benchmarks, security audits, and codebase changes are analyzed in real-time and written as structured intelligence into 22 Notion databases. A Health agent synthesizes everything into a single score, while Claude AI generates narratives, risk assessments, and actionable recommendations.

Quick Start
1Configure

Run ENGRAM and open the dashboard. The first-start wizard walks you through connecting Notion, GitHub, and Claude.

2Initialize

The setup wizard creates all 22 Notion databases with schemas, relations, and sample data automatically.

3Webhooks

4Monitor

Data flows in via webhooks. Intelligence layers analyze and write to Notion. Dashboard updates automatically.

Intelligence Layer Reference
⚖

Decisions

Tracks RFCs from draft to implementation. Detects drift between approved architecture and actual code. AI-generated impact analysis and trade-off summaries.

⚡

Pulse

Ingests CI benchmark results from Criterion, Hyperfine, k6. Detects regressions using rolling statistical baselines with configurable thresholds.

⚠

Shield

Processes audit output from cargo-audit, npm-audit, pip-audit, OSV-Scanner. Auto-triages CVEs by CVSS. AI fix recommendations.

☰

Atlas

Maps codebase modules with AI summaries, complexity scores, dependency graphs. Identifies knowledge gaps, generates onboarding tracks.

🔒

Vault

Inventories env vars, API keys, secrets. Tracks rotation schedules, detects missing credentials, alerts on policy violations.

🔍

Review

AI PR review with blocker/suggestion/nit classification. Detects recurring patterns, enforces playbook rules, generates tech debt items.

❤

Health

Aggregates all layers into a weighted composite score (0-100). Generates weekly digests with AI narrative, risks, wins, actions.

🕑

Timeline

Records every engineering event into a unified, searchable timeline. Marks milestones, tracks project evolution.

📦

Release

Aggregates PR, RFC, CVE, regression data for release readiness. AI release notes, migration guides, go/no-go recommendations.
